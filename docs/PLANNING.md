# Love Letter - 기획 문서

> 연인끼리 매일 편지를 주고받는 서비스

## 1. 프로젝트 개요

### 1.1 목표
연인 간의 소통을 더 깊고 의미있게 만드는 디지털 편지 교환 플랫폼

### 1.2 핵심 가치
- **진정성**: 빠른 메시지가 아닌, 정성스러운 편지
- **기대감**: 예약 공개로 편지를 기다리는 설렘
- **추억**: 주고받은 편지를 보관하고 다시 볼 수 있음
- **꾸준함**: 매일 리마인더로 소통 습관 형성

### 1.3 타겟 사용자
- 장거리 연애 커플
- 바쁜 일상 속 소통이 필요한 커플
- 특별한 기념일을 챙기고 싶은 커플

## 2. 핵심 기능

### 2.1 편지 작성 및 전송
| 기능 | 설명 |
|------|------|
| 텍스트 편지 | 긴 글 작성 가능 |
| 사진 첨부 | 이미지 업로드 |
| 음성 메시지 | 녹음 첨부 |
| 동영상 첨부 | 짧은 영상 업로드 |
| 테마/편지지 | 다양한 디자인 선택 |
| 즉시 전달 | 작성 즉시 상대방에게 전달 |
| 예약 공개 | 특정 시간에 편지가 열림 |

### 2.2 커플 연결
| 방식 | 설명 |
|------|------|
| 초대 코드 | 고유 코드 생성 → 상대방 입력 |
| 이메일 초대 | 이메일로 초대 링크 발송 |
| 전화번호 검색 | 전화번호로 사용자 찾기 |
| 닉네임 검색 | 닉네임으로 사용자 찾기 |

### 2.3 알림 시스템
| 알림 | 설명 |
|------|------|
| 새 편지 도착 | 상대방이 편지를 보냄 |
| 편지 공개 | 예약된 편지가 열림 |
| 매일 리마인더 | 설정한 시간에 편지 쓰기 알림 |
| 기념일 알림 | 등록한 기념일 알림 |

### 2.4 보관 및 기록
| 기능 | 설명 |
|------|------|
| 편지 보관함 | 모든 편지 아카이브 |
| 검색 | 날짜, 키워드로 검색 |
| 기념일 관리 | 기념일 등록 및 D-day |

## 3. 사용자 시나리오

### 시나리오 1: 첫 사용 - 커플 연결
```
1. 회원가입 (이메일 또는 소셜 로그인)
2. 프로필 설정 (닉네임, 알림 시간)
3. 초대 코드 생성 또는 입력
4. 상대방과 연결 완료
5. 첫 편지 작성 유도
```

### 시나리오 2: 일상 - 편지 주고받기
```
1. 매일 저녁 9시 리마인더 알림
2. 앱 열어서 편지 작성
3. 테마 선택, 사진 첨부
4. 즉시 전송 또는 예약 공개 설정
5. 상대방이 편지 읽음
6. 답장 작성
```

### 시나리오 3: 특별한 날 - 기념일 편지
```
1. 기념일 등록 (100일, 1주년 등)
2. 기념일 며칠 전 알림
3. 특별한 편지 작성 (예약 공개로 자정에 오픈)
4. 기념일 당일 편지 공개
```

### 시나리오 4: 추억 - 편지 다시 보기
```
1. 보관함 진입
2. 월별/연도별 필터
3. 특정 편지 선택
4. 그때의 편지 다시 읽기
```

## 4. 비기능 요구사항

### 4.1 성능
- 페이지 로드: 2초 이내
- 편지 전송: 3초 이내
- 미디어 업로드: 10MB 이하 파일

### 4.2 보안
- HTTPS 필수
- 편지 내용 암호화
- 인증 토큰 관리

### 4.3 가용성
- 99% 이상 가동률
- Serverless로 자동 스케일링

### 4.4 접근성
- 모바일 우선 반응형 디자인
- 다크모드 지원
